# Туториал: Telegram Mini App с нуля через Claude Code

**Аудитория:** полный новичок — не знаком ни с Claude Code, ни с разработкой Telegram Mini App
**Формат:** пошаговый туториал с готовыми промптами для копирования
**Язык:** русский
**Пример приложения:** трекер собаки (профиль, вес, уход, кормление, здоровье, команды, дневник, фото-дневник)
**Стек:** React + Vite, все данные в localStorage, без бэкенда
**Результат:** работающее приложение на своём домене с автодеплоем при изменениях

---

## Шаг 0. Введение

### Что построим
Приложение для ухода за собакой:
- Профиль питомца (фото, имя, порода, возраст, вес с графиком)
- Еженедельный трекер ухода (зубы, когти, купание)
- Расписание кормления
- Журнал здоровья (вакцины, осмотры, операции)
- Обработка от паразитов с напоминаниями
- Трекер команд с прогресс-барами
- Дневник (похвалы и жалобы)
- Фото-дневник в виде календаря

Все данные хранятся в localStorage браузера — ничего не уходит в интернет, работает офлайн.

### Что понадобится
- Компьютер с macOS, Windows или Linux
- Аккаунт на [github.com](https://github.com) (бесплатно)
- VPS-сервер и домен (~$5–10/мес)
- Подписка Claude Pro или API-ключ Anthropic

### Что такое Claude Code
Claude Code — это инструмент командной строки от Anthropic. Ты описываешь что хочешь сделать обычными словами, а Claude пишет код, запускает команды и решает проблемы самостоятельно. Тебе не нужно знать программирование — только объяснять задачу.

## Шаг 1. Установка Claude Code

### Установка Node.js
Перед установкой Claude Code нужен Node.js. Скачай с [nodejs.org](https://nodejs.org) версию LTS и установи.

Проверь в терминале:
```
node --version
```
Должно вывести что-то вроде `v20.x.x`.

### Установка Claude Code
Открой терминал и выполни:
```
npm install -g @anthropic-ai/claude-code
```

### Первый запуск и логин
```
claude
```
Claude Code попросит войти через браузер — следуй инструкциям на экране.

### Проверочный промпт
После входа убедись что всё работает:
```
Привет! Скажи что ты умеешь делать в двух предложениях.
```

## Шаг 1.5. Настройка агентов и работа с дизайном

### Что такое агенты
Вместо одного Claude на все задачи удобнее использовать специализированных агентов: `/designer` думает как дизайнер, `/frontend-dev` как разработчик интерфейса. Это даёт более точные и качественные результаты.

### Установка Superpowers
```
claude plugin install superpowers@claude-plugins-official
```

Проверь доступные агенты:
```
/help
```

Основные агенты для нашего проекта:
- `/designer` — продумывает UI, компоненты, состояния
- `/frontend-dev` — реализует экраны, компоненты, логику

### Работа со скриншотами дизайна
Claude видит изображения прямо в терминале. Если у тебя есть макет — нарисованный от руки, в Figma, Keynote или на бумаге — просто прикрепи его:

```
[прикрепи файл скриншота]

```

Советы:
- Чем чище и детальнее скриншот — тем точнее результат
- Можно набросать макет от руки и сфотографировать
- Можно прислать несколько скриншотов сразу для сравнения

## Шаг 3. Разработка по экранам

После того как Claude создал скаффолд, реализуй каждый экран по очереди. Один экран за раз — так легче проверять результат и давать фидбек.

### Принцип работы
1. Отправь промпт для одного экрана
2. Подожди пока Claude реализует его
3. Открой в браузере и проверь
4. Если что-то не так — опиши словами что изменить
5. Переходи к следующему экрану

### Промпты по экранам

**Экран: Главная**
```
/frontend-dev Реализуй экран "Главная" полностью:
- шапка с фото питомца (заглушка по умолчанию), именем, породой, возрастом и обратным отсчётом до дня рождения
- блок веса с мини-графиком (используй recharts или chart.js) и кнопкой добавить замер
- трекер ухода: зубы (цель 4–7), когти (цель 1), купание (цель 1), 7 кружков по дням недели
- расписание кормления с выделением текущего и ближайшего приёма
- список записей здоровья с цветовыми статусами
- блок паразитов с предупреждением за 7 дней и при просрочке
Все данные читать/писать в localStorage.
```

**Экран: Команды**
```
/frontend-dev Реализуй экран "Команды":
- список команд с прогресс-баром 0–100%
- кнопки − и + меняют уровень на 10%
- статусы в зависимости от уровня: 0–20% «Начинаем», 30–50% «Учится», 60–80% «Хорошо», 90–100% «Отлично»
- кнопка добавить команду (название + начальный уровень)
- свайп или кнопка удалить команду
Данные в localStorage.
```

**Экран: Дневник**
```
/frontend-dev Реализуй экран "Дневник":
- две вкладки: Похвалы 🌟 и Жалобы 😤
- в каждой вкладке список записей (текст + дата)
- кнопка добавить запись (textarea + дата)
- кнопка удалить запись
- пустое состояние когда записей нет
Данные в localStorage.
```

**Экран: Фото-дневник**
```
/frontend-dev Реализуй экран "Фото-дневник":
- календарь по месяцам, переключение вперёд/назад

- фото хранится в localStorage как base64
- в ячейке дня показывается миниатюра фото
- если несколько фото — показывает счётчик (+2 и т.д.)
```

**Экран: Профиль**
```
/frontend-dev Реализуй экран "Профиль":
- форма: имя питомца, порода, дата рождения, пол (кнопки М/Ж)
```

## Шаг 6. Деплой через Claude

### Подготовка SSH-ключа

SSH-ключ — это способ подключиться к серверу без пароля. Проверь есть ли он уже:

```
Проверь есть ли у меня SSH-ключ (~/.ssh/id_rsa.pub или ~/.ssh/id_ed25519.pub) и покажи публичный ключ.
```

Если ключа нет, Claude создаст его:
```
Создай SSH-ключ для подключения к серверу.
```

Скопируй публичный ключ (он выглядит как длинная строка начинающаяся с `ssh-rsa` или `ssh-ed25519`).

### Добавление ключа на сервер
При создании VPS в панели провайдера будет поле "SSH key" — вставь туда скопированный публичный ключ. Если сервер уже создан, добавь ключ через панель управления провайдера.

### Проверочный промпт
Сначала убедись что Claude может подключиться:
```
Подключись к серверу [IP-адрес] как root по SSH и скажи что там установлено (версия Ubuntu, свободное место на диске).
```

### Промпт для деплоя
```
Зайди на сервер [IP-адрес] как root по SSH.
Задеплой React-приложение из текущей папки:
1. Установи Node.js и nginx если не установлены
2. Собери приложение командой npm run build
3. Скопируй содержимое папки dist на сервер
4. Настрой nginx для раздачи статических файлов
5. Убедись что приложение открывается по IP-адресу сервера
```

## Шаг 7. Привязка домена

### Добавление A-записи
1. Зайди в панель управления своего регистратора доменов
2. Найди раздел DNS или "Управление DNS"
3. Добавь A-запись:
   - **Имя/Host:** `@` (означает корневой домен)
   - **Значение/Value:** IP-адрес твоего сервера
   - **TTL:** 3600 (или оставь по умолчанию)
4. Если хочешь чтобы работал `www.mydomain.com` — добавь такую же A-запись с именем `www`

DNS обновляется за 5–30 минут (иногда до 24 часов).

### Промпт для настройки nginx и SSL
```
Зайди на сервер [IP-адрес] как root.
Настрой nginx для домена [mydomain.com]:
1. Обнови конфиг nginx для обработки запросов по имени домена
2. Установи certbot и получи бесплатный SSL-сертификат через Let's Encrypt
3. Настрой автоматическое перенаправление с http на https
4. Проверь что сайт открывается по https://mydomain.com
```

## Шаг 7.5. Настройка Telegram Mini App

Сейчас приложение — просто сайт. Чтобы оно открывалось прямо из Telegram как Mini App, нужно создать бота и привязать к нему ссылку на сайт.

### Создание бота в BotFather
1. Открой Telegram и найди [@BotFather](https://t.me/BotFather)
2. Отправь `/newbot`
3. Придумай имя бота (например `My Dog Tracker`) и username (например `mydogtracker_bot`)
4. BotFather выдаст токен вида `123456789:AAF...` — сохрани его

### Настройка кнопки меню
1. В BotFather отправь `/mybots`
2. Выбери своего бота
3. Bot Settings → Menu Button → Configure menu button
4. Вставь URL твоего сайта: `https://mydomain.com`
5. Введи название кнопки, например `Открыть`

Теперь в чате с ботом появится кнопка, которая открывает твоё приложение как Mini App внутри Telegram.

### Проверка
Открой своего бота в Telegram и нажми кнопку меню — должно открыться приложение.

## Шаг 8. CI/CD — автодеплой

Теперь настроим автоматический деплой: при каждом изменении кода и пуше в GitHub — приложение автоматически обновится на сервере.

### Промпт для настройки CI/CD
```
Настрой GitHub Actions для автодеплоя:
1. При каждом push в ветку main — собирать проект и деплоить на сервер [IP-адрес] через SSH
2. Создай файл .github/workflows/deploy.yml
3. После создания файла объясни пошагово как добавить SSH-ключ в GitHub Secrets (Settings → Secrets → Actions), чтобы GitHub Actions мог подключиться к серверу
```

### Добавление секрета в GitHub
Claude объяснит этот шаг, но в общем это выглядит так:
1. Открой репозиторий на GitHub
2. Settings → Secrets and variables → Actions
3. New repository secret
4. Имя: `SSH_PRIVATE_KEY`, значение: содержимое файла `~/.ssh/id_ed25519` (приватный ключ)

После этого каждый `git push` будет автоматически обновлять сайт.
