# Туториал: Telegram Mini App с нуля через Claude Code

**Аудитория:** полный новичок — не знаком ни с Claude Code, ни с разработкой Telegram Mini App
**Формат:** пошаговый туториал с готовыми промптами для копирования
**Язык:** русский
**Пример приложения:** трекер собаки (профиль, вес, уход, кормление, здоровье, команды, дневник, фото-дневник)
**Стек:** React + Vite, все данные в localStorage, без бэкенда
**Результат:** работающее приложение на своём домене с автодеплоем при изменениях

---

## Шаг 0. Введение

### Что построим
Приложение для ухода за собакой с пятью разделами: профиль питомца, трекер ухода и кормления, журнал здоровья, трекер команд, фото-дневник. Все данные хранятся прямо в браузере — ничего не уходит в интернет, работает офлайн. В конце приложение будет доступно по ссылке и откроется как Mini App прямо в Telegram.

### Что понадобится
- Компьютер с macOS, Windows или Linux
- Аккаунт на [github.com](https://github.com) (бесплатно)
- VPS-сервер и домен (~$5–10/мес)
- Подписка Claude Pro

### Что такое Claude Code
Claude Code — это инструмент командной строки от Anthropic. Ты описываешь что хочешь сделать обычными словами, а Claude пишет код, запускает команды и решает проблемы самостоятельно. Тебе не нужно знать программирование — только объяснять задачу.

## Шаг 1. Установка Claude Code

### Как открыть терминал
Терминал — это окно где вводятся текстовые команды. Именно через него запускается Claude Code.

- **Mac:** нажми `Cmd+Пробел`, введи `Terminal`, открой
- **Windows:** нажми `Win+R`, введи `cmd`, нажми Enter (или найди `PowerShell` в меню Пуск)
- **Linux:** `Ctrl+Alt+T`

### Установка Node.js
Перед установкой Claude Code нужен Node.js. Скачай с [nodejs.org](https://nodejs.org) версию LTS и установи.

Проверь в терминале:
```
node --version
```
Должно вывести что-то вроде `v20.x.x`. Если команда не найдена — перезапусти терминал после установки.

### Установка Claude Code
Открой терминал и выполни:
```
npm install -g @anthropic-ai/claude-code
```

### Первый запуск и логин
```
claude
```
Claude Code попросит войти через браузер — следуй инструкциям на экране.

### Проверочный промпт
После входа убедись что всё работает:
```
Привет! Скажи что ты умеешь делать в двух предложениях.
```

## Шаг 1.5. Настройка skills и работа с дизайном

### Что такое skills
Skills — это специализированные роли для Claude. Вместо одного Claude на все задачи ты вызываешь нужного специалиста — например `/frontend-developer-skill` отвечает как опытный фронтенд-разработчик и даёт более точные результаты чем обычный чат.

Skills можно найти и установить через маркетплейс — команда `/skills` в Claude Code открывает каталог.

### Установка skills

**Superpowers** — набор специализированных ролей (дизайнер, разработчик, ревьюер и др.):
```
claude plugin install superpowers@claude-plugins-official
```

**frontend-developer-skill** — универсальный фронтенд-разработчик:
```
claude skill install 404kidwiz/claude-supercode-skills
```

Проверь установленные skills:
```
/skills
```

Основной skill для нашего проекта:
- `/frontend-developer-skill` — реализует экраны, компоненты, логику по скриншоту или текстовому описанию

## Шаг 4. GitHub — репозиторий

### Что такое репозиторий и зачем он нужен
Репозиторий — это хранилище кода твоего проекта в интернете. Он нужен по двум причинам:
- **Резервная копия** — код хранится не только у тебя на компьютере, его не потеряешь
- **Автодеплой** — в шаге 8 мы настроим так, что при каждом изменении кода сайт обновляется автоматически

GitHub — самый популярный сервис для хранения репозиториев, бесплатный.

### Создание аккаунта и репозитория
1. Зарегистрируйся на [github.com](https://github.com)
2. Нажми `+` → `New repository`
3. Назови репозиторий, например `my-dog-app`
4. Выбери `Private` или `Public`
5. Нажми `Create repository`
6. Скопируй URL репозитория (вида `https://github.com/username/my-dog-app.git`)

### Промпт для первого пуша
```
Инициализируй git репозиторий, сделай первый коммит со всеми файлами проекта и запушь в этот репозиторий: https://github.com/username/my-dog-app.git
```

Claude выполнит `git init`, `git add`, `git commit` и `git push`.

## Шаг 5. Покупка сервера и домена

### Сначала: подготовь SSH-ключ
При создании сервера провайдер сразу попросит SSH-ключ — это твой электронный "пропуск" для подключения к серверу. Подготовь его заранее.

Открой новый терминал (не закрывая Claude Code) и спроси Claude:
```
Проверь есть ли у меня SSH-ключ (~/.ssh/id_rsa.pub или ~/.ssh/id_ed25519.pub) и покажи публичный ключ.
```

Если ключа нет:
```
Создай SSH-ключ для подключения к серверу.
```

Скопируй публичный ключ — длинную строку начинающуюся с `ssh-rsa` или `ssh-ed25519`. Он понадобится через минуту.

### Что такое сервер и домен
**Сервер** — это компьютер в интернете, который работает круглосуточно и отдаёт твоё приложение всем кто открывает ссылку. Без сервера приложение живёт только у тебя на компьютере. Мы арендуем VPS (virtual private server) — это твой личный кусочек такого компьютера за небольшую ежемесячную плату.

**Домен** — это адрес сайта, например `mydogapp.com`. Без домена приложение доступно только по числовому IP-адресу вида `185.12.34.56` — некрасиво и неудобно. Домен покупается один раз в год.

### Где купить VPS-сервер

Рекомендуемые провайдеры:
- **Hetzner** (hetzner.com) — лучшее соотношение цена/качество, серверы в Европе, от €4/мес
- **DigitalOcean** (digitalocean.com) — удобный интерфейс, от $6/мес
- **REG.ru** — российский провайдер, оплата картой РФ, от 200 руб/мес

Минимальный тариф: 1 vCPU, 1GB RAM — достаточно для нашего приложения.

**При создании сервера:** выбери Ubuntu 22.04. Провайдер попросит добавить SSH-ключ — об этом в шаге 6.

### Где купить домен

- **Namecheap** (namecheap.com) — от $1–10/год, международный
- **REG.ru** — российский регистратор, удобно если сервер тоже там
- **Cloudflare** (cloudflare.com/products/registrar) — по себестоимости

Выбери имя домена и купи его. Запомни где ты его купил — это понадобится в шаге 7.

## Шаг 6. Деплой через Claude

SSH-ключ ты уже подготовил в шаге 5 и добавил при создании сервера — теперь Claude может подключиться к нему напрямую.

### Проверочный промпт
Сначала убедись что Claude может подключиться:
```
Подключись к серверу [IP-адрес] как root по SSH и скажи что там установлено (версия Ubuntu, свободное место на диске).
```

### Промпт для деплоя
```
Зайди на сервер [IP-адрес] как root по SSH.

1. Установи Node.js и nginx если не установлены
2. Собери приложение командой npm run build
3. Скопируй содержимое папки dist на сервер
4. Настрой nginx для раздачи статических файлов
5. Убедись что приложение открывается по IP-адресу сервера
```

## Шаг 7. Привязка домена

### Добавление A-записи
1. Зайди в панель управления своего регистратора доменов
2. Найди раздел DNS или "Управление DNS"
3. Добавь A-запись:
   - **Имя/Host:** `@` (означает корневой домен)
   - **Значение/Value:** IP-адрес твоего сервера
   - **TTL:** 3600 (или оставь по умолчанию)
4. Если хочешь чтобы работал `www.mydomain.com` — добавь такую же A-запись с именем `www`

DNS обновляется за 5–30 минут (иногда до 24 часов).

### Промпт для настройки nginx и SSL
```
Зайди на сервер [IP-адрес] как root.
Настрой nginx для домена [mydomain.com]:
1. Обнови конфиг nginx для обработки запросов по имени домена
2. Установи certbot и получи бесплатный SSL-сертификат через Let's Encrypt
3. Настрой автоматическое перенаправление с http на https
4. Проверь что сайт открывается по https://mydomain.com
```

## Шаг 7.5. Настройка Telegram Mini App

Сейчас приложение — просто сайт. Чтобы оно открывалось прямо из Telegram как Mini App, нужно создать бота и привязать к нему ссылку на сайт.

### Создание бота в BotFather
1. Открой Telegram и найди [@BotFather](https://t.me/BotFather)
2. Отправь `/newbot`
3. Придумай имя бота (например `My Dog Tracker`) и username (например `mydogtracker_bot`)
4. BotFather выдаст токен вида `123456789:AAF...` — он нам не понадобится (нет бэкенда), можно проигнорировать

### Настройка кнопки меню
1. В BotFather отправь `/mybots`
2. Выбери своего бота
3. Bot Settings → Menu Button → Configure menu button
4. Вставь URL твоего сайта: `https://mydomain.com`
5. Введи название кнопки, например `Открыть`

Теперь в чате с ботом появится кнопка, которая открывает твоё приложение как Mini App внутри Telegram.

### Проверка
Открой своего бота в Telegram и нажми кнопку меню — должно открыться приложение.

## Шаг 8. CI/CD — автодеплой

Теперь настроим автоматический деплой: при каждом изменении кода и пуше в GitHub — приложение автоматически обновится на сервере.

### Промпт для настройки CI/CD
```
Настрой GitHub Actions для автодеплоя:
1. При каждом push в ветку main — собирать проект и деплоить на сервер [IP-адрес] через SSH
2. Создай файл .github/workflows/deploy.yml
3. После создания файла объясни пошагово как добавить SSH-ключ в GitHub Secrets (Settings → Secrets → Actions), чтобы GitHub Actions мог подключиться к серверу
```

### Добавление секрета в GitHub
Сначала получи содержимое приватного ключа — спроси Claude:
```
Покажи содержимое моего приватного SSH-ключа (~/.ssh/id_ed25519).
```
Скопируй весь вывод (от `-----BEGIN` до `-----END`).

Затем добавь его в GitHub:
1. Открой репозиторий на GitHub
2. Settings → Secrets and variables → Actions
3. New repository secret
4. Имя: `SSH_PRIVATE_KEY`, значение: вставь скопированный ключ

После этого каждый `git push` будет автоматически обновлять сайт.
